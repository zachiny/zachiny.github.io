使用npm安装hexo搭建github个人博客

 2021-04-27发表  顾师傅  应用教程 7 分钟读完 (大约1056个字)

(教程)[nodejs|hexo|github]

本文章讲述在安装nodejs环境后，再安装Hexo等安装包搭建一个Linux服务器。

  • 顾师傅三步总结：
    第一步： 安装nodejs相关版本包并配置。
    第二步： 下载安装hexo包及相关包并初始化。
    第三步： 创建github仓库修改相应hexo配置文件即可上传。

安装nodejs相关版本包并配置

官网：https://nodejs.org
官网编译包地址：https://nodejs.org/dist/
根据实际使用需要，下载对应平台的软件后安装。

安装:
wget https://nodejs.org/dist/v14.0.0/node-v14.0.0-linux-x64.tar.gz
如果出现
【-bash: wget: command not found】
则
【yum install -y wget】

解压：

目录结构如下:
node-v14.0.0-linux-x64/
├── bin
│ ├── node
│ ├── npm -> ../lib/node_modules/npm/bin/npm-cli.js
│ └── npm -> ../lib/node_modules/npm/bin/npx-cli.js
├── CHANGELOG.md
├── include
│ └── node
├── lib
│ └── node_modules
├── LICENSE
├── README.md
└── share
├── doc
├── man
└── systemtap

注: “XXX/bin/node”、“XXX/bin/npm”为绝对路径

当上面执行成功，则：
node --version
输出
v14.0.0

若网络速度较慢，可使用npm国内淘宝镜像

  1. 命令
    [root@server ~]# npm config set registry https://registry.npm.taobao.org
  2. 验证命令
    [root@server ~]# npm config get registry
    https://registry.npm.taobao.org/
    如果返回https://registry.npm.taobao.org ，说明镜像配置成功。

下载安装hexo包及相关包并初始化

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

  • Hexo包
    官网：https://hexo.io/
    文档：https://hexo.io/docs/

安装 Hexo 前置环境：
yum install git-core
安装 Hexo：
npm install -g hexo-cli
此时可发现前面node文件下多了
├── bin
│   ├── hexo -> ../lib/node_modules/hexo-cli/bin/hexo
同上创建软链接
ln -s XXX/bin/hexo /usr/local/bin/

  • 初始化建站 若第一步执行失败：
    [Error: ENOENT: no such file or directory, scandir ‘/XXX/node-v14.0.0-linux-x64/lib/node_modules/hexo-cli/assets’]
    则替换为执行：
    git clone https://gitee.com/gitmirror/hexo-starter.git

[可选]

  • 使用hexo主题icarus

[\可选]

生成配置：
hexo g
启动服务：
hexo s

输出信息：
INFO Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

服务器IP加端口号4000即可访问Hexo博客

创建github仓库修改相应hexo配置文件即可上传

一、创建仓库
注册/登录GitHub账号后，打开 https://github.com/new 页面创建仓库，仅填写”Repository name”此项即可。

注: 仓库名称一定要符合《 用户名.github.io 》的格式


二、SSH登录仓库

执行命令生成密钥对 :
ssh-keygen -C "邮箱地址"
一直按enter回车
在“~/.ssh/”目录下，生成了公钥id_rsa.pub和私钥id_rsa。
执行cat ~/.ssh/id_rsa.pub
将输出的公钥内容复制到
https://github.com/settings/ssh/new的key位置。

输入命令，绑定GitHub账户:

测试连接

ssh -T git@github.com

提示含 《You’ve successfully authenticated》，表明已经成功连接。

三、部署Hexo到GitHub

Ctrl + C中断后台运行的Hexo博客
安装发布到github需要的npm包
npm install hexo-deployer-git --save
修改当前目录下的_config.yml文件。
vim _config.yml
最下面修改为

分别执行：
hexo g
hexo d
打开https://用户名.github.io即可看到部署的博客

[可选]
安装Hexo写文章的插件

npm install --save hexo-admin
hexo server -d

访问 http://localhost:4000/admin/ 即可编辑

使用Vagrant创建服务器

 2021-04-21发表  顾师傅  应用教程 4 分钟读完 (大约601个字)

(教程)[Oracle VM VirtualBox|Vagrant|Linux]

本文章讲述在VirtualBox环境下，使用Vagrant安装Linux服务器。

  • 顾师傅三步总结：
    第一步： 下载安装Oracle VM VirtualBox环境。
    第二步： 下载安装Vagrant软件及想要的镜像。
    第三步： 执行相应Vagrant命令，服务器即可启动。

下载安装Oracle VM VirtualBox

官网：https://www.virtualbox.org/
下载链接：https://www.virtualbox.org/wiki/Downloads
https://www.virtualbox.org/wiki/Downloads

根据实际使用需要，下载对应平台的软件后安装。

下载安装Vagrant软件及想要的镜像

  • ❶ Vagrant软件
    官网：https://www.vagrantup.com/
    下载链接：https://www.vagrantup.com/downloads

  • ❷ Vagrant镜像
    镜像即系统。
    根据自身使用习惯，选择对应系统的版本。 https://app.vagrantup.com/boxes/search

限于各种下载问题，特此贴出几个国内镜像。

Centos

下载链接：https://mirrors.ustc.edu.cn/centos-cloud/centos

镜像例子: https://mirrors.ustc.edu.cn/centos-cloud/centos/7/vagrant/x86_64/images/CentOS-7.box

Ubuntu

下载链接：https://mirrors.tuna.tsinghua.edu.cn/ubuntu-cloud-images/bionic/current/

镜像例子: https://mirrors.tuna.tsinghua.edu.cn/ubuntu-cloud-images/bionic/current/bionic-server-cloudimg-amd64-vagrant.box

执行相应Vagrant命令，服务器即可启动

  • 两款软件都安装好之后，新建一个自定义的文件夹，以便统一存放之后的虚拟机。

    D:\vagrant 下目录结构：

  • 执行添加镜像，启动虚拟服务器。

    vagrant添加镜像

    vagrant box add server_1 D:\vagrant\CentOS-7.1.1503-x86_64.box

    语法：vagrant box add 【自定义虚拟机名称】【镜像位置】
    vagrant初始化镜像

    vagrant init server_1

    语法：vagrant init 【自定义虚拟机名称】
    vagrant启动镜像

    vagrant up

语法：vagrant up

至此，虚拟机设置并启动完毕。

设置连接服务器

命令：vagrant ssh
使用vagrant连接虚拟机
命令：sudo passwd root
使用sudo命令设置root密码
输入2次密码设置root用户便可登录

此时使用Oracle VM VirtualBox便可登录，不过由于操作较为不便，可设置使用自己喜欢的连接工具设置连接。

  • 注：端口号为初始化时端口号。
